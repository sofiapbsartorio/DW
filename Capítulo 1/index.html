Capítulo 1

1.1 Quais são os 3 tipos de códigos básicos de uma página de hipertexto? Explique para que serve cada um dos tipos.

Resposta: HTML, CSS e JavaScript. O HTML serve para definir quais elementos serão exibidos em uma página. Já o CSS é uma linguagem usada para definir os estilos dos elementos e conteúdos de uma página, ou seja, como eles serão mostrado. Por último, o JavaScript, que é uma linguagem de programação e é utilizada em sites que possuem interação com o usuário e/ou efeitos visuais mais avançados que podem ser feitos com o uso de JavaScript.

1.4 Descreva a diferença entre uma página de hipertexto estática e uma dinâmica, pontuando as vantagens e desvantagens de cada um dos dois tipos.

Resposta: Um site é estático quando suas páginas são totalmente compostas por arquivos prontos. Quando as páginas são geradas por uma aplicação web, então o site é dito dinâmico.

1.5 Qual é o caminho percorrido por uma requisição HTTP a uma página de hipertexto estática?

Resposta: Primeiro, o usuário informa uma URL ao navegador. A partir do nome do domínio, o navegador consulta um servidor DNS e retorna o IP do domínio consultado. Uma vez que o IP tenha sido obtido, uma requisição HTTP é feita ao servidor web correspondete ao IP em questão, e o recurso solicitado é devolvido.

1.6 Qual é o caminho percorrido por uma requisição HTTP a uma página de hipertexto dinâmica?

Resposta: Primeiro, o usuário informa uma URL ao navegador. A partir do nome do domínio, o navegador consulta um servidor DNS e retorna o IP do domínio consultado. Uma vez que o IP tenha sido obtido, uma requisição HTTP é feita ao servidor web correspondete ao IP em questão. No caso de páginas dinâmicas, alguns dados a mais são necessários, e esses dados costumam ficar armazenados em um banco de dados. A aplicação web combina esses dados com um modelo genérico de página HTML e envia de volta ao navegador através de uma resposta HTTP.

1.7 Quais as vantagens da atualização parcial de conteúdo em segundo plano em uma página de hipertexto usando JavaScript?

Resposta: Carregando os conteúdos em segundo plano, pode-se atualizar partes da página nem a necessidade de requisitar toda a página novamente, fazendo com que a experiência do usuário seja a melhor possível e diminuindo consideravelmente a sobrecarga do servidor.
